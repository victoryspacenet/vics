# 빅토리스페이스(VICS) 최종점검용 테스트 시나리오

**문서 목적**: 테스트 배포·스테이징 환경에서 **페이지(라우트) 단위**로 실제 클릭·입력 검증을 수행할 때 사용하는 체크리스트와 **통과 기준**을 정리합니다.  
**대상**: QA 담당, 기획/운영, 개발자 스모크 테스트  
**기준일**: 2026-04-26 (라우트 기준: `src/App.jsx`)

---

## 1. 테스트 전 공통 준비

| 항목 | 내용 |
|------|------|
| 환경 | 스테이징 URL, Supabase 스테이징 프로젝트, `.env` / Netlify 환경변수가 **운영과 분리**되어 있는지 확인 |
| 브라우저 | Chrome 최신 + 모바일 Safari(또는 iOS Chrome) 각 1회 이상 |
| 계정 | (A) 비로그인, (B) 일반 사용자, (C) 티어 다양(bronze/silver/gold/diamond 가능 시), (D) 관리자(운영자) |
| 데이터 | 실제와 유사한 매치업 **진행 중 / 마감 직전 / 종료** 샘플, 댓글·투표 데이터 1건 이상 |
| 기록 | 이슈 발생 시 **URL, 계정 종류, 재현 순서, 스크린샷/콘솔 에러**를 함께 남김 |

**공통 통과 기준(모든 화면)**

- [ ] 화면이 **빈 흰 화면·무한 로딩** 없이 의도된 레이아웃으로 표시된다.
- [ ] 콘솔에 **치명적 에러**(빨간 Uncaught, 반복되는 401/403 루프)가 없다. (경고만 있는 경우는 별도 기록)
- [ ] 하단 네비·헤더와 **본문이 겹치지 않고**, 주요 버튼이 **한 손엄지 영역**에서 눌린다(모바일).
- [ ] 뒤로가기·새로고침 후에도 **라우트·로그인 상태**가 일관되게 유지되거나, 명시된 리다이렉트만 발생한다.
- [ ] 토스트/모달 문구가 **사용자에게 다음 행동**을 알 수 있다.

## 2. 크로스컷팅 시나리오 (페이지를 가로지르는 검증)

### 2.1 인증·세션

| ID | 시나리오 | 통과 기준 |
|----|----------|-----------|
| X-AUTH-01 | 비로그인 상태에서 투표/댓글/마이페이지 등 제한 기능 클릭 | 로그인 모달 또는 로그인 페이지로 유도되고, 로그인 후 **의도한 화면으로 복귀**할 수 있다(OAuth 시 `PostOAuthRedirect` 동작 포함). |
| X-AUTH-02 | 로그인 유지 중 탭을 장시간 방치 후 복귀 | 알림/실시간 등 **핵심 기능이 복구**되거나(재구독), 한 번 새로고침으로 정상화된다. |
| X-AUTH-03 | 로그아웃 | 로그인 상태가 해제되고, 민감 화면 접근 시 다시 로그인 요구가 된다. |

### 2.2 데이터·저장소 원칙(프로젝트 규칙)

| ID | 시나리오 | 통과 기준 |
|----|----------|-----------|
| X-DATA-01 | 공지·문의·팝업·제재 등 **운영 데이터** 조회/저장 | 브라우저 **localStorage에 비밀/운영 데이터를 저장하지 않는다**(개발자 도구 Application 탭으로 샘플 확인). Supabase 기준 동작. |

### 2.3 보안·접근

| ID | 시나리오 | 통과 기준 |
|----|----------|-----------|
| X-SEC-01 | 일반 사용자로 `/admin` 하위 URL 직접 입력 | 권한 없으면 **접근 차단·리다이렉트** 등 정책대로 동작한다. |
| X-SEC-02 | `/restricted` 등 제한 사용자 플로우 | 안내 문구와 함께 **이탈·문의 경로**가 명확하다. |

### 2.4 성능·UX

| ID | 시나리오 | 통과 기준 |
|----|----------|-----------|
| X-PERF-01 | 메인 피드·매치업 목록 첫 로딩 | LCP 수준에서 **과도한 지연 없이** 스켈레톤/콘텐츠가 나타난다(3G Fast 기준은 팀 합의). |
| X-A11Y-01 | `prefers-reduced-motion` 켠 상태(가능 시) | 애니메이션이 과하지 않고 **핵심 정보는 동일하게** 읽힌다. |

## 4. 회귀 테스트 묶음 (릴리즈 직전 30분 코스)

다음 순서로 **한 번에** 돌리면 핵심 리스크를 빠르게 덮을 수 있습니다.

1. 비로그인: `/` → 카드 클릭 → 투표 시도 → 로그인 모달  
2. 로그인: `/feed/best` 스크롤 → `/matchup/:id` 투표  
3. 종료 매치업: 댓글 1건 + 답글 1건(골드 계정이면 이모지 바)  
4. `/mypage` → `/mypage/edit` 저장 → `/mypage/edit/image`  
5. `/notice` → 공지 1건 열람  
6. `/inquiry` → FAQ 1건 → 문의 폼 초기화만(제출은 스테이징 정책에 따라)  
7. 관리자: `/admin/dashboard` 접속 후 공지/문의 중 **1개 메뉴만** 저장 플로우 확인  

**전체 통과**: 위 각 단계에서 블로킹 이슈 0건, P1(중요) 이슈는 문서화되어 있고 릴리즈 판단이 내려진 상태.
